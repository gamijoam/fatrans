## VIOLACIONES CRÍTICAS (🔴)

### 1. **[CRÍTICA] Hibernate ddl-auto=validate impide creación automática de tablas**

**Archivo:** `backend/src/main/resources/application.yml:12`

```yaml
jpa:
  hibernate:
    ddl-auto: validate
```

**Descripción:**  
El modo `validate` de Hibernate **solo verifica** que el esquema existente coincida con las entidades, pero **no crea ni modifica** ninguna tabla. Esto es apropiado para producción, pero significa que en un entorno nuevo las tablas simplemente no existen.

**Impacto:**  
- La aplicación inicia sin errores
- Cualquier operación de base de datos falla con `Table 'cuentas_ahorro' does not exist`
- Los 3 módulos (auth, socios, ahorros) son completamente inoperantes

**Corrección:**

Cambiar a `update` para desarrollo (crea/actualiza tablas automáticamente):

```yaml
jpa:
  hibernate:
    ddl-auto: update
```

O usar `create-drop` para desarrollo limpio (crea al iniciar, elimina al detener):

```yaml
jpa:
  hibernate:
    ddl-auto: create-drop
```

> **NOTA:** Para producción, usar `validate` con migraciones Flyway/Liquibase es lo recomendado.
