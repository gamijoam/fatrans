### 🟡 RUNTIME-01: Posible NPE en `SocioEntity.toDomain()` — Colección Roles Null

**Archivo:** `backend/src/main/java/com/tufondo/socios/infrastructure/persistence/entity/SocioEntity.java`  
**Línea aproximada:** 175

```java
// ❌ PROBLEMA: this.roles puede ser null si no se cargó la colección
public Socio toDomain() {
    return Socio.builder()
        // ...
        .roles(this.roles)  // ← this.roles puede ser null (JPA puede retornar null para ElementCollection vacío)
        .eliminado(this.eliminado)
        .build();
}
```

**Impacto:** Si JPA retorna `null` para una `@ElementCollection` vacía (en lugar de una colección vacía), `Socio.roles` será `null`, causando `NullPointerException` en operaciones posteriores.

**Corrección:**
```java
public Socio toDomain() {
    return Socio.builder()
        // ...
        .roles(this.roles != null ? this.roles : Set.of())
        .eliminado(this.eliminado)
        .build();
}
```

## 4. MEJORAS RECOMENDADAS

### 🟡 MEJORA-01: `ElementCollection` con `FetchType.EAGER` en `SocioEntity`

**Archivo:** `backend/src/main/java/com/tufondo/socios/infrastructure/persistence/entity/SocioEntity.java`  
**Línea:** ~130

```java
// ❌ ACTUAL: EAGER causa N+1 queries en listados
@ElementCollection(fetch = FetchType.EAGER)
@CollectionTable(name = "socio_roles", joinColumns = @JoinColumn(name = "socio_id"))
@Column(name = "rol")
private Set<String> roles;
```

**Recomendación:** Cambiar a `LAZY` y usar Entity Graph si se necesitan los roles frecuentemente:
```java
@ElementCollection(fetch = FetchType.LAZY)
@Fetch(FetchMode.SUBSELECT)  // Para evitar N+1 en listados
@CollectionTable(name = "socio_roles", joinColumns = @JoinColumn(name = "socio_id"))
@Column(name = "rol")
private Set<String> roles;
```
