# AUDITORÍA EXHAUSTIVA - MÓDULO AUTH

## 1. RESUMEN EJECUTIVO

Se auditaron **14 archivos** del módulo de autenticación (`com.tufondo.auth`). El módulo presenta una estructura general adecuada siguiendo Clean Architecture, sin embargo se detectaron **violaciones críticas** relacionadas con pérdida de datos en el mapeo entidad-dominio y queries inconsistentes. También hay problemas de **inconsistencia** en el manejo de enums y parámetros sin uso.

---

## 2. VIOLACIONES CRÍTICAS

### 🔴 CRÍTICA 1: Pérdida de datos en `SesionEntity.aDominio()`

**Archivo:** `backend/src/main/java/com/tufondo/auth/infrastructure/persistence/entity/SesionEntity.java`  
**Líneas:** 84-90

```java
public com.tufondo.auth.domain.model.Sesion aDominio() {
    return new com.tufondo.auth.domain.model.Sesion(
            this.id,
            this.usuarioId,
            this.refreshToken,
            this.accessTokenExpiracion,
            this.refreshTokenExpiracion
    );
}
```

**Problema:** El modelo de dominio `Sesion` solo recibe 5 parámetros en su constructor, pero la entidad `SesionEntity` almacena campos adicionales: `activo`, `tipoToken`, `fechaCreacion`, `ultimaActividad`. **Estos datos se pierden al convertir a dominio.**

**Impacto:** La información de actividad, tipo de token y estado activo se persiste en DB pero nunca se recupera al dominio.

---

### 🔴 CRÍTICA 3: Parámetro `ipAddress` sin uso en query

**Archivo:** `backend/src/main/java/com/tufondo/auth/infrastructure/persistence/jpa/SesionJpaRepository.java`  
**Líneas:** 35-37

## 3. VIOLACIONES DE ARQUITECTURA

### 🟠 VIOLACIÓN 1: Mapeo incompleto de `Sesion` Domain → Entity

**Archivo:** `backend/src/main/java/com/tufondo/auth/infrastructure/persistence/entity/SesionEntity.java`  
**Líneas:** 93-103

```java
public static SesionEntity desdeDominio(com.tufondo.auth.domain.model.Sesion sesion) {
    entity.setId(sesion.id());
    entity.setUsuarioId(sesion.usuarioId());
    // ...
    // Defaults hardcodeados:
    entity.setActivo(true);
    entity.setTipoToken("BEARER");  // ⚠️ ¿Por qué hardcodear?
    entity.setFechaCreacion(Instant.now());
    entity.setUltimaActividad(Instant.now());
    return entity;
}
```

**Problema:** El dominio `Sesion` no tiene campos `activo`, `tipoToken`, `fechaCreacion`, `ultimaActividad`. Los está asignando con valores por defecto, pero **estos datos existen en la DB**.

**Recomendación:** Extender el modelo de dominio `Sesion` para incluir estos campos, o reconsiderar si pertenecen al dominio.

---

### 🟠 VIOLACIÓN 2: Tipo `Object` para parámetro `rol` en JPA Query

**Archivo:** `backend/src/main/java/com/tufondo/auth/infrastructure/persistence/jpa/UsuarioJpaRepository.java`  
**Líneas:** 34-38

```java
@Query("SELECT u FROM UsuarioEntity u WHERE u.rol = :rol AND u.cuentaActiva = true")
List<UsuarioEntity> findByRolAndCuentaActivaTrue(@Param("rol") Object rol);

@Query("SELECT u FROM UsuarioEntity u WHERE u.socioId = :socioId AND u.rol = :rol AND u.cuentaActiva = true")
Optional<UsuarioEntity> findBySocioIdAndRolAndCuentaActivaTrue(@Param("socioId") UUID socioId, @Param("rol") Object rol);
```

**Problema:** Usar `Object rol` en lugar de `Rol rol` es incorrecto. Debería usar el tipo `Rol` directamente.

### 🟠 VIOLACIÓN 3: Inconsistencia entre tipo `TipoToken` y columna String

**Archivo:** `backend/src/main/java/com/tufondo/auth/domain/model/enums/TipoToken.java`

**Archivo:** `SesionEntity.java`

```java
// CAMBIAR DE:
```
