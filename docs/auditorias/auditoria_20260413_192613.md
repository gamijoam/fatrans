## 2. VIOLACIONES CRÍTICAS

### 🔴 BUG #1: `@Transactional` incorrecto en método de solo lectura

**Archivo:** `backend/src/main/java/com/tufondo/auth/infrastructure/persistence/adapter/SesionRepositoryImpl.java`

**Línea:** 73-75

```java
@Override
@Transactional  // ❌ INCORRECTO - Este método solo lee datos
public int contarSesionesActivasPorUsuario(UUID usuarioId) {
    return jpaRepository.contarSesionesActivasPorUsuario(usuarioId);
}
```

**Problema:** El método `contarSesionesActivasPorUsuario()` es unicamente de lectura (SELECT COUNT), pero está anotado con `@Transactional` sin `readOnly = true`. Esto genera开销 innecesario de transacción de escritura.

**Corrección:**
```java
@Override
@Transactional(readOnly = true)
public int contarSesionesActivasPorUsuario(UUID usuarioId) {
    return jpaRepository.contarSesionesActivasPorUsuario(usuarioId);
}
```

### 🔴 BUG #3: Posible NullPointerException en flujo de stream

**Archivo:** `backend/src/main/java/com/tufondo/auth/infrastructure/persistence/adapter/SesionRepositoryImpl.java`

**Línea:** 41-44

```java
@Override
@Transactional(readOnly = true)
public List<Sesion> buscarSesionesActivasPorUsuario(UUID usuarioId) {
    return jpaRepository.findSesionesActivasPorUsuario(usuarioId)
            .stream()
            .map(SesionEntity::aDominio)
            .toList();
}
```

**Problema:** Si `jpaRepository.findSesionesActivasPorUsuario()` retorna `null` (en lugar de una lista vacía), el `.stream()` lanzaría `NullPointerException`.

**Corrección recomendada:**
```java
@Override
@Transactional(readOnly = true)
public List<Sesion> buscarSesionesActivasPorUsuario(UUID usuarioId) {
    List<SesionEntity> entidades = jpaRepository.findSesionesActivasPorUsuario(usuarioId);
    if (entidades == null) {
        return List.of();
    }
    return entidades.stream()
            .map(SesionEntity::aDominio)
            .toList();
}
```

## 3. VIOLACIONES DE ARQUITECTURA

### 🟠 VIOLACIÓN #1: Inconsistencia en normalización de email

**Archivos:**
- `backend/src/main/java/com/tufondo/auth/infrastructure/persistence/adapter/UsuarioRepositoryImpl.java`
- `backend/src/main/java/com/tufondo/auth/infrastructure/persistence/jpa/UsuarioJpaRepository.java`

**Líneas afectadas:**
- `UsuarioRepositoryImpl.java` línea 38: `correo.toLowerCase().trim()`
- `UsuarioRepositoryImpl.java` línea 47: `correo.toLowerCase().trim()`
- `UsuarioJpaRepository.java` línea 24: No normaliza en query

**Problema:** 
- En `UsuarioRepositoryImpl.buscarPorCorreoElectronico()` se normaliza el email a minúsculas y se recorta espacios.
- Pero en `UsuarioJpaRepository.findByCorreoElectronico()` se usa directamente el valor sin normalizar.
- Esto puede causar inconsistencias si se llama directamente al JPA Repository.

**Diagrama del problema:**
```
buscarPorCorreoElectronico("USUARIO@Ejemplo.COM")
        │
        ▼
UsuarioRepositoryImpl (normaliza) → "usuario@ejemplo.com"
        │
        ▼
UsuarioJpaRepository.findByCorreoElectronico("usuario@ejemplo.com")
        │
        ▼
¿Pero si alguien llama directo al JPA?
UsuarioJpaRepository.findByCorreoElectronico("USUARIO@Ejemplo.COM")
        │
        ▼
❌ No encuentra el usuario (case-sensitive)
```

### 🟠 VIOLACIÓN #4: Acoplamiento en mapeo de entidades

**Archivo:** `backend/src/main/java/com/tufondo/auth/infrastructure/persistence/entity/UsuarioEntity.java`

**Línea:** 54-70

```java

    entity.id = usuario.id();  // ❌ Dependencia bidireccional
    // ... setea todos los campos

}
```

**Problema:** La entidad tiene conocimiento del modelo de dominio `Usuario`. En Clean Architecture puro, este mapeo debería estar en el **Mapper/Adapter layer**, no en la entidad misma.

**Arquitectura esperada:**
```
┌─────────────────────────────────────────┐
│         Domain (Sin dependencias)        │
│  ┌─────────────────────────────────┐     │
│  │  Usuario.java                   │     │
│  │  (No conoce a UsuarioEntity)    │     │
│  └─────────────────────────────────┘     │
└─────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────┐
│    Infrastructure (Implementación)      │
│  ┌─────────────────────────────────┐     │
│  │  UsuarioEntity.java             │     │
│  │  (Mapeo básico, sin lógica)     │     │
│  └─────────────────────────────────┘     │
│  ┌─────────────────────────────────┐     │
│  │  UsuarioEntityMapper.java       │     │ ← Debería estar aquí
│  │  (Conversión Dominio ↔ Entidad) │     │
│  └─────────────────────────────────┘     │
└─────────────────────────────────────────┘
```

**Corrección recomendada:** Mover `desdeDominio()` y `aDominio()` a un mapper dedicado.
